## 1. 需求分析

### 1.1 项目背景

本课程综合训练项目旨在深入理解 Linux 系统中进程管理的基本概念，包括进程的定义、状态、控制块（PCB）等。通过实现进程控制、进程调度和进程同步三大核心模块，掌握操作系统的核心机制。

| 模块 | 功能 | 实现方式 |

| 进程调度 | FCFS + Round Robin | 手动实现调度算法 |
| 进程同步 | 文件下载并发限制 | Semaphore 信号量 |

## 2. 原理分析

### 2.1 进程控制

#### fork() 原理
`fork()` 系统调用用于创建一个新进程（子进程），子进程是父进程的副本。
- 父进程返回子进程 PID
- 子进程返回 0
- 采用写时拷贝（Copy-on-Write）技术

**Windows 适配**：由于 MinGW 不完全支持 POSIX 的 `fork()`，本实现使用 `CreateThread()` 创建线程来模拟父子进程关系。

#### exec() 原理
`exec()` 系列函数用新程序替换当前进程映像。
- 调用成功不返回
- 失败返回 -1

**Windows 适配**：使用 `CreateProcess()` 创建新进程执行 `cmd.exe /c dir` 命令。

#### wait() 原理
`wait()` 使父进程等待子进程终止，回收子进程资源，防止僵尸进程。

**Windows 适配**：使用 `WaitForSingleObject()` 等待线程结束，`GetExitCodeThread()` 获取退出码。

### 2.2 进程调度

#### FCFS（先来先服务）
- **策略**：按进程到达顺序依次执行
- **特点**：非抢占式，实现简单
- **缺点**：可能出现"护航效应"（长进程阻塞短进程）

**计算公式**：
- 完成时间 = 开始时间 + 执行时间
- 周转时间 = 完成时间 - 到达时间
- 等待时间 = 周转时间 - 执行时间

#### Round Robin（时间片轮转）
- **策略**：每个进程轮流执行固定时间片
- **特点**：抢占式，公平性好
- **参数**：时间片 quantum = 2

**执行逻辑**：
1. 按到达时间排序
2. 循环遍历就绪队列
3. 每次执行 min(剩余时间, 时间片)
4. 直到所有进程完成

### 2.3 进程同步

#### 信号量（Semaphore）
- **P 操作（Wait）**：申请资源，信号量减1；若为0则阻塞
- **V 操作（Signal）**：释放资源，信号量加1；唤醒等待线程

**Windows 实现**：使用 `CreateSemaphore()` 和 `WaitForSingleObject()` / `ReleaseSemaphore()`

#### 下载并发限制问题
- **场景**：最多允许 N 个线程同时下载
- **初始化**：`CreateSemaphore(NULL, 5, 5, NULL)` — 初始值5，最大值5
- **下载前**：WaitForSingleObject（P操作）
- **下载后**：ReleaseSemaphore（V操作）

#### 生产者-消费者问题
- **缓冲区**：容量为 BUFFER_SIZE 的循环队列
- **三个信号量**：
  - mutex：互斥访问缓冲区
  - not_full：空槽位数（初始 = BUFFER_SIZE）
  - not_empty：满槽位数（初始 = 0）
